Introduction

Unit 1 covers the basics of writing and executing code in Julia. This lesson introduces you to the Julia programming language and it's basic constructs. Targeted towards users who have not used Julia language in the past, minimal programming language experience is reuired. As such, we will not be going to dive too deeply into the 'why' things are done, but focus on the 'how' things are done.

In order to begin programming in Julia you will need some way to execute your source code.  There are two ways to do this: interactively on the command line, or from a file with the runtime interpreter.  The first is good for prototyping and learning, while the second is what you will use to run the code in production.

REPL

The REPL is the command line interactive interface for Julia; it stands for  Read-Eval-Print-Loop, and is used to execute code interactively.  When you develop and execute Julia code you will do so by writing Julia source files, and executing then in your  terminal.  The REPL is a handy tool for just trying out small snippets of code.  In the first unit we will use the REPL to demonstrate the introductory concepts.  Later in the course you will execute code non-interactively in script mode.

When inside the REPL the built in documentation can be accessed by prepending a command the '?' symbol.  For example, if you type ?print you will get the help page for the built in print function.

To start an interactive session, open up your terminal and type Julia on the command line, you should see the following. To exit simply type exit(). If you are executing this code in the Graham cluster environment, you will find a link at the bottom of the page for loading modules.

Start the julia REPL in the terminal.

The REPL starts in the default Julian prompt mode. In the REPL you will notice that the last value computed is printed out, end a line with a semicolon ';' to suppress this behaviour. This last value is also stored in the variable 'ans'. Most console programs will allow you to clear the screen with [CTRL]-L. 


            julia> 2 + 1
            3

            julia> ans
            3

            julia> 3 + 5;
Test out simple operations in the REPL.

You can enter shell mode by pressing the ';' key when the cursor is at the beginning of a line. This mode will allow you to enter a single shell command, then it returns to the default Julian prompt. If you want to enter multiple commands you can always start a new shell session with 'bash' (or 'sh). The shell prompt, shown below, indicates you are in shell mode. 


            julia> ;
            shell> ls /
            bin   dev  home  lib    lib64   media  opt   root  sbin  srv  temp  usr
            boot  etc  init  lib32  libx32  mnt    proc  run   snap  sys  tmp   var
Upon typing ';' the prompt changes to 'shell>'

Your first script

Just to warm up we will create the classic "Hello World" program. Use vim (or your preferred command line text editor) to create a 'helloWorld.jl' source file. In this file put the following into the file and save it. 

            println("Hello World!");

To execute this script 

            julia helloWorld.jl

You should see the result. 

            Hello World!

You can run scripts on demand in the REPL by using the include keyword. This will execute all the commands in the script as if you had typed them in. Using include twice may overwrite variables you have already used. 

            julia> include("helloworld.jl")
            Hello World
Include is the same as typing the code.

Comments

Single line comments start with the '#' symbol and run to the end of line. Multi-line comments start with th '#=' symbol and run until the '=#' symbol. 
            # single line comment
            #= multi-line
               comment
            =#
Comments are ignored by the interpreter.
